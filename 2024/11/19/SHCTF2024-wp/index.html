# [Week1] 有 WiFi 干嘛不用呢？

题目给了一堆文件

还不知道有啥用，wireshark 看看 01.cap 文件看到这四个包，是 WiFi 建立连接的四次握手

然后在网上搜寻资料下一步要爆破密码，看到 may 文件夹里面有很多类似密码的文件

把这些文件名输出到一个文件作为字典爆破密码

可是发现没找到，这不对吧，再看看文件里面的内容，发现每个文件内容都是方括号包裹的一串可能是密码的字符串

写个脚本把这些字符都提取出来

然后得到新的字典，再爆破一次发现成功爆破到了密码

参考博客：https://cloud.tencent.com/developer/article/2072385

一套检查发现 idat 数据块有问题， tweakpng 检查一下（题目也有提示挨打（idat））

发现是一张 4995 为底，一张 5185 为底的两张图片叠在一起，删掉前面三个 idat

发现后面那张图片显示出来了得到 flag

# [Week2] 练假成真

二维码缺失手工补齐

首先用到 crazybox 工具来辅助分析二维码

首先运用 QRazyBox 的 Data Sequence Analysis 的功能，以便更方便的读取 qrcode 每个块的数据信息

可以看到，选中最右下角的块，显示 0100

根据编码表知道这是 8-bits byte 的编码方式

接着是上面那块 这记录了这个二维码总共存储了多少个字符 如图所示 是 31 个

再上一块 就是存储数据的块了 我们需要从这里开始修复 首先是补齐 flag 头: SHCTF {(01010011 01001000 01000011 01010100 01000110 01111011)

然后将二进制的值按块直接复制进去回车就能修改

然后接着补齐 flag 头

再把剩下部分都填为未知

最后就能通过 Reed-Solomon Decoder 还原出二维码所存数据，得到 flag

但是解密中间部分发现是 fake F1agf4K3rF1agf4K3r

010 再看这张图片发现还有藏一张图片在里面将其分离出来后发现还是一张二维码 不过这回是完整的了 直接扫码就能得到一个 Base64 表:**「ABCDjp0yIJKLSVOPQNMzURWX9cabZdefgFiEklmnohqrstuvwxhTG21345678Y+/」** 用这个表解密 F1agf4K3rF1agf4K3r 就能得到 THis_F14GTHis_F14G
