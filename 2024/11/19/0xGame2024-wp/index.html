# [Week 3] cargo_shop

题目源码

看到源码，这是一个 rust 写的简单的购物系统，先看到 flag 路由，如果买了名为 flag 的 good 访问就给出环境变量 FLAG ，如果没有就提示 you need to buy the flag goods to get flag

题目提示 i32 和 u32，这是 rust 语言里面两种数据类型

i32 有符号 32 位整数，表示从 -2,147,483,648 到 2,147,483,647 的值
u32 无符号 32 位整数，表示从 0 到 4,294,967,295 的值

看到源码里

在购买和卖出时的计算表示是 i32. 由此可以想到利用 i32 到 u32 类型转换时的溢出

直接购买一个超过上限的数量

发现初始 1000 块钱可以买这么多个

并且余额还有 980 哈哈哈，直接暴富了，然后再把拥有的卖出去，直到能够买 flag，再访问 flag 路由就能得到 flag（这里要注意，购买的数量要在 2,147,483,647 和 4,294,967,295 之间，否则会触发 u32 的 if 判断，提示余额不足）

# [Week 3] hello_jwt

源码如下

可以看到有三个路由，先看到 register 路由，注册一个账号，然后跳转到 login 路由，然后再登录跳转到 flag 路由提示 Only admin can view the flag，看到源码，是对 token 做了验证，验证 role 参数是否是 admin，这时候看到下面还有两个路由，hint1 和 hint2，猜测是根据 hint1 和 hint2 的提示得到 jwt 的密钥 key 来完成加密

那么就先看看 hint1，解密 token 再根据判断条件把 role 改成 Please, give me the hint ，密钥为空

加密得到 token，bp 改包

得到第一个提示，猜测要爆破密钥

再看 hint2，操作同 hint1（注意 hint2 是有 key 的，我就说怎么发半天包都是 invalid token 😠）

确定了是要爆破，那就用工具跑个字典，（五位小写字母）

得到了 key 是 zrajz ，再回到 flag 路由，改 role 为 admin ，key 设置为 zrajz ，改包得到 flag

# [Week 2] 互联网的一角

题目给了一个静态网页，提示 CNAME 记录

CNAME 记录是用于将一个域名映射到另一个域名的 DNS 记录。它通常用于处理复杂的域名解析，比如将 www 子域名解析到主域名，或者将不同的服务指向不同的主机名。

总的来说就是将一个域名解析到另一个域名， ping 命令或者 nslookup 都可以查看 dns 解析记录

dns 记录种类如下：

  • A 记录：将域名解析为 IPv4 地址。
  • AAAA 记录：将域名解析为 IPv6 地址。
  • CNAME 记录：别名记录，将一个域名映射到另一个域名。
  • MX 记录：邮件交换记录，用于指示电子邮件服务器。
  • TXT 记录：可以包含任意文本数据，常用于验证和身份验证目的。
  • NS 记录：指定负责解析该域名的权威 DNS 服务器。

可以看到在 ping 时解析到了 oxg4me2024.github.io 这个域名，然后在 github 上面搜索这个用户 id，在他的 github 仓库里面找到 flag

但是是 fake，再翻看历史记录看到 flag
