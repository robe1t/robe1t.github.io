# 每周大挑战 1

题目 hint 是 spring 信息泄露

先随便输点东西在输入框抓个包看看是不是有可以注入啥的，然后尝试着爆破一下（爆破字典）弱口令密码，爆破到 admin 123456

是可以登陆进去的，但是什么东西都没有–

那就 dirsearch 扫描一下，扫到了很多目录，一个一个看

在 env 目录里面直接能看到 flag（非预期解）

有一个 heapdump 目录，打开直接下载了一个名为 heapdump 的文件，然后百度搜一下，找了个工具！！image-20240521221707710

把工具和文件放同一个目录下面

启动工具查看这个文件

然后看到了里面有一个 key，是 shiro 框架的密钥（由于这个 key 是自定义的所以爆破不出来）

再打开 shiro 反序列化 balabala 工具，放进这个 key

就可以做命令执行了，然后 ls 看到 fllllag

cat 一下这个文件拿到 flag

# 每周大挑战 2

开局一个 jar 包

看了看 hint 去搜了反编译 jar 包

下载 jadx 反编译工具（下载链接：https://github.com/skylot/jadx）

上网搜了下 idea 也能反编译，于是下了个插件

（好像是 idea 自带的？？？插件名字：Java Bytecode Decompiler ）

反编译参考博客利用 intellij idea 工具如何反编译.jar_jar 包反编译 - CSDN 博客

然后反编译一下 jar 包解压在 idea 里面就能打开，目录一路往下翻看到这个 shiroconfig 文件里面有 rememberMe_2024 的字段，然后提交 question1，flag 正确

打开 jadx 工具，在里面搜索 rememberme 字段但是没看到有 key 的相关东西，把类名加上，再搜索看到了 DEFAULT_CIPHER_KEY_BYTES 的字段，后面还跟了一段看着像 base64 编码的字符串，那八九不离十就是加密的 key 了，复制过去提交显示 flag 正确

参考博客：https://chenxi9981.github.io/RealWorld/

接着复制博客中的代码过来

新建这两个类，然后运行（注意 class 文件的地址要改成自己电脑上的路径）

再新建 exp.py 终端运行一下 (去文件所在位置开终端，在 idea 里面的终端自带换行没法直接复制使用)

!!image-20240528235138755

然后在 hackbar 里面添加一个 cooki

在攻击机里监听端口并拿到 flag
